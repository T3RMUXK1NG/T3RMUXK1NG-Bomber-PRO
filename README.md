## ⚙️ Configuration

When you run the bomber, you'll be asked:

1. **Country Code** — Without the + sign (e.g., 91 for India)
2. **Target Number/Email** — The target's phone number or email
3. **Number of Attacks** — How many SMS/Calls/Mails to send
4. **Delay Time** — Delay between attacks in seconds
5. **Thread Count** — Number of parallel threads (recommended: count/10)

### Tips for Best Performance
- Use **single thread** with considerable delay time for best results
- Always use the **latest version** of RS Bomber PRO
- Ensure **active internet connection**
- Do **not** use VPN — it can increase fail rates

## ❓ FAQ

### Is there any official website/app?
No. The only official releases are on **GitHub** and **PyPI**. If you find it elsewhere, it's not official.

### Poor Internet Connection Detected?
- Check your internet connection
- Make sure `openssl` is installed
- Try pinging any remote site
- Try reinstalling if nothing works

### Which countries are supported?
- **SMS**: Most countries supported (delivery rate varies)
- **Calls**: India only
- **Mail**: Universal

### Why is the limit so low?
Due to the amount of requests, the APIs can die. To prevent a bigger outtake, limits are in place.

### How to protect yourself?
Use OTP blockers and activate DND (Do Not Disturb).

### Why does it fail sometimes?
Due to overuse, a bunch of APIs have been taken offline. It's normal if you don't receive all messages.
